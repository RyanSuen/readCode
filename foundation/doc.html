一、使用Foundation

无论您使用的是Scss还是CSS我们都能很轻松地开始一个项目，这样就可以帮助你减少制作响应式网站的时间。 Foundation强大的CSS预处理器，可以帮助你编写更干净，更有组织性的CSS，你可以更轻松地维护CSS。我们已经写了强大的JavaScript插件，这将使网页更容易实现跨屏幕浏览。

1、Foundation是什么？

我们已经拥有了设计响应式网站涉及到的有用功能：

网格

我们的网格几乎支持在任何设备上按顺序偏移，这将使布局变得很简单!

排版

该类型是基于一个黄金比例的模块，它可以很方便的更新Scss或者定制我们的CSS。

按钮

按钮是网络的一个核心交互式元件。我们已经包含了用于创建简单应用程序的工具栏风格。

导航栏

通常人们会在网页上通过导航栏查看页面。而在Foundation的导航栏风格中包含一个复杂顶部导航栏，以及支持3级下拉的简单导航栏、侧边栏和子导航。

表单

内置的基础网格，您可以快速的创建简单或复杂的表单，并且很容易验证自定义选项，单选按钮等。

UI元素

我们还有很多有用的元素类如：visibility（隐藏或显示基于屏幕的大小或方向的东西）、labels、alerts、tooltips、panels等。

Section选项卡

无论是整个页面或简单的选项卡的内容，都可以使用基础标签完成。不仅美观而且交互性很强。

Orbit幻灯

一个强大的图像、内容的幻灯片显示插件，其中常用的选项包括：such as timers, paddles, bullet thumbs等。

Reveal弹出层

您可以很容易地调用Reveal弹出层，实现跨设备支持。

Clearing幻灯

使用简单的无序列表就可以打造可心的全屏幻灯内容了。

Magellan

灵活的跟随屏幕移动并能识别位置的导航条，你只需要设置好tag位置或将其关闭。

Joyride

这个插件可以让你您的网站把您的应用程序显示给用户，Joyride可以轻松地自定义使用CSS或SCSS变量。

tips工具

tooltips可以让你快速添加hover、click等事件的tips元素。

下拉菜单

我们已经制作了一个下拉菜单插件，可以很容易地将弹出层下拉到网页上的任何位置。

2、Block Grid

block grid会把你的内容均匀分布在网格内。如果你想建立5个连续的图像，而不管屏幕大小都能均匀缩放，那么block grid就非常适合你。

2.1 预定义HTML类

Block grids是有ul.small-block-grid-#或者ul.large-block-grid-#组成。如果您使用small-block-grid，无论屏幕大小，网格都会保持间距和配置。如果您使用large-block-grid，列表会堆叠在小型设备上。如果灵活的使用这些类，你可以为每个断点的布局进行控制和配置。

<!-- Using only the small-block-grid -->
<ul class="small-block-grid-2">
  <li><img src="../img/demos/demo1.png"></li>
  <li><img src="../img/demos/demo2.png"></li>
  <li><img src="../img/demos/demo3.png"></li>
  <li><img src="../img/demos/demo4.png"></li>
</ul>

<!-- Using only the large-block-grid -->
<ul class="large-block-grid-4">
  <li><img src="../img/demos/demo1.png"></li>
  <li><img src="../img/demos/demo2.png"></li>
  <li><img src="../img/demos/demo3.png"></li>
  <li><img src="../img/demos/demo4.png"></li>
</ul>

<!-- Using both block grids together for different layouts -->
<ul class="small-block-grid-2 large-block-grid-4">
  <li><img src="../img/demos/demo1.png"></li>
  <li><img src="../img/demos/demo2.png"></li>
  <li><img src="../img/demos/demo3.png"></li>
  <li><img src="../img/demos/demo4.png"></li>
</ul>

如果想要这些样式生效，请确保您有默认的基础CSS包或Block grids的自定义包。这里有我们的默认的HTML页面结构。

2.2 使用mixin

我们的block-grids包含了SCSS的mixin.要使用mixin, 你需要从Github安装扩展或者提取_foundation-global.scss, _global.scss 和 _block-grid.scss到你的项目内. 从那里,你可以在你自己的SCSS顶部导入文件，像这样：

@import "foundation/foundation-global", "foundation/components/global", "foundation/components/block-grid";

如果你要使用mixin, 你可能需要包含该样式到你喜欢的class或ID上, 只要确保您按照我们的结构进行标记.你可以使用任何你想要的样式，只需更改配置文件即可。

<ul class="your-class-name">
  <li><img src="../img/demos/demo1.png"></li>
  <li><img src="../img/demos/demo2.png"></li>
  <li><img src="../img/demos/demo3.png"></li>
  <li><img src="../img/demos/demo4.png"></li>
</ul>

2.3 Quick Mixin

你可以利用我们的全局mixin建立自己的block-grid，包括您自定义样式表中的class和ID。该mixin包含快速配置block-grid布局的选项。全局mixin看起来像这样：

/* Mixin and configurable options */
.your-class-name { @include block-grid($per-row, $spacing, $base-styles); }

/* This controls how many elements will be on each row of the block grid. */
/* Set this to whatever number you need, up to the max allowed in the variable */
$per-row: false

/* This controls how much space is between each item in the block grid */
$spacing: $block-grid-default-spacing

/* This controls whether or not base styles come through, set to false to leave out */
/* This is handy for when you create multiple block-grids and want less CSS output repetition */
$base-style: true

2.4 Clearing Block Grid Rows

这段代码会清除N+1的网格的样式，代码看起来像这样：

.your-small-class-name > li { clear: none !important; }

Note: If you are using the same class across breakpoints, your mixin should override this and you won't need it. If you want to control the layout using different classes, use the above snippet.

2.5 可用的SCSS变量

/* We use this to control the maximum blocks and spacing */
$block-grid-elements: 12;
$block-grid-default-spacing: 10px;

/* Enables media queries for block-grid classes. Set to false if writing semantic HTML. */
$block-grid-media-queries: true
